# Ultra-Low Latency Server тАФ Design Spec

## Goal

Achieve sub-1ms (<0.5ms target) end-to-end latency for the `/predict` endpoint by replacing the Python stack with a Rust server using ONNX Runtime for model inference.

## Architecture

```
HTTP Request тЖТ Actix-Web (Rust) тЖТ serde JSON parse тЖТ State (HashMap<IP, VecDeque>) тЖТ Feature Engineering (Rust) тЖТ StandardScaler (Rust, hardcoded params) тЖТ ONNX Runtime (C++) тЖТ JSON Response
```

## Components

### 1. ONNX Export Script (`ultra-low-latency/export_onnx.py`)

Python script that runs once to convert trained artifacts:

- Loads `artifacts/model.joblib` and `artifacts/scaler.joblib`
- Exports Isolation Forest to `ultra-low-latency/server/assets/model.onnx` via `skl2onnx`
- Exports scaler parameters (mean, scale arrays) to `ultra-low-latency/server/assets/scaler_params.json`
- Validates ONNX output matches scikit-learn output on a test input

### 2. HTTP Server (`server/src/main.rs`)

- Actix-Web with `/predict` (POST), `/health` (GET), `/` (GET)
- Request/response structs match existing API contract (same JSON fields)
- State and ONNX session stored in actix `web::Data`

### 3. State Management (`server/src/state.rs`)

- `HashMap<String, VecDeque<RequestRecord>>` behind a `Mutex`
- `add_record()`: append + cleanup records older than 300s
- `get_records()`: filter by timestamp window (30s, 300s)
- Same logic as `api/state.py`, direct port

### 4. Feature Engineering (`server/src/features.rs`)

- Port of `lib/feature_engineering.py` тАФ 10 features
- `entropy()`, `std_dev()`, `compute_features()` in pure Rust
- Known UA list hardcoded as a static `HashSet`

### 5. Inference (`server/src/inference.rs`)

- StandardScaler: `(x - mean) / scale` using params from `scaler_params.json`, loaded at startup
- ONNX Runtime via `ort` crate: load `model.onnx` at startup, run inference per request
- Returns anomaly score + prediction (-1/1)

### 6. Schemas (`server/src/schemas.rs`)

- `PredictRequest`, `PredictResponse` as serde structs
- Same fields as current `api/schemas.py`

## API Contract (unchanged)

**POST /predict**

Request:
```json
{
  "ip": "192.168.1.100",
  "timestamp": 1751500000.0,
  "endpoint": "/login",
  "method": "POST",
  "status_code": 401,
  "payload_size": 256,
  "user_agent": "python-requests/2.31",
  "response_time": 45.0
}
```

Response:
```json
{
  "ip": "192.168.1.100",
  "is_anomaly": false,
  "anomaly_score": -0.1234,
  "top_features": {"request_count_30s": 1.0, ...},
  "timing_ms": {"state_update": 0.01, "feature_calc": 0.02, ...}
}
```

## Dependencies

### Python (export only)
- `skl2onnx`, `onnxruntime` (for validation)

### Rust
- `actix-web` тАФ HTTP server
- `serde`, `serde_json` тАФ JSON serialization
- `ort` тАФ ONNX Runtime bindings
- `tokio` тАФ async runtime (required by actix-web)

## Latency Budget

| Stage | Target |
|---|---|
| HTTP parse + routing | < 0.05 ms |
| State update + query | < 0.02 ms |
| Feature engineering | < 0.05 ms |
| Scaler + ONNX inference | < 0.15 ms |
| Response serialization | < 0.03 ms |
| **Total** | **< 0.3 ms** |

## Validation

- Run export script, verify ONNX output matches scikit-learn within 1e-6
- Build and run Rust server
- Run same 100-request latency benchmark as baseline
- Compare mean/p50/p95/p99 against Python baseline (10.5ms)
